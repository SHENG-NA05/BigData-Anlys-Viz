# 圖書館智慧策展系統 - 系統規格書 (System Specification)

本系統規格書為「圖書館智慧策展系統」之技術實作標準，供 UI、Web Service、DB、AI 與 Testing 開發人員遵循使用。

---

## 1. 系統架構與技術棧

- **前端 (UI)**：React (v18+) + Vite + TailwindCSS + Ant Design (v5+)
- **後端 (Web Service)**：Python (v3.10+) + FastAPI
- **資料庫 (DB)**：PostgreSQL (v14+) + pgvector 擴充套件 + Alembic (資料庫版本遷移) + SQLAlchemy (ORM)
- **AI 引擎 (AI)**：Google Gemini API (Gemini 1.5 Flash/Pro & Text Embedding API) / OpenRouter
- **測試 (Testing)**：PyTest (後端單元測試) + Jest (前端元件測試)

## 3. 資料庫 Schema 設計 (SQL DDL)

請 DB 開發人員在 PostgreSQL 中執行以下 DDL 腳本以建置資料表。本專案啟用 `pgvector` 進行書目與主題的向量相似度搜尋：

```sql
-- 啟用向量資料庫擴充套件
CREATE EXTENSION IF NOT EXISTS vector;

-- 1. 使用者角色與帳號表
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(100) UNIQUE NOT NULL,
    hashed_password VARCHAR(255) NOT NULL,
    role VARCHAR(50) NOT NULL DEFAULT 'curator', -- 'admin', 'curator'
    sso_provider VARCHAR(50) NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- 2. 圖書館館藏圖書表
CREATE TABLE catalog_books (
    id SERIAL PRIMARY KEY,
    title VARCHAR(255) NOT NULL,                    -- 書名
    isbn VARCHAR(50) NOT NULL,                      -- ISBN 碼
    classification_no VARCHAR(100) NOT NULL,        -- 中文圖書分類法分類號 (例如 733.4)
    summary TEXT NULL,                              -- 書籍簡介 (供 AI 比對匹配用)
    embedding vector(768) NULL,                      -- 書籍簡介與標題之 768 維向量 (Gemini text-embedding-004)
    imported_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_catalog_books_isbn ON catalog_books(isbn);
CREATE INDEX idx_catalog_books_class_no ON catalog_books(classification_no);
-- 建立 HNSW 向量索引以加速 Cosine 距離搜尋
CREATE INDEX idx_catalog_books_embedding ON catalog_books USING hnsw (embedding vector_cosine_ops);

-- 3. AI 歷史策展主題大綱表
CREATE TABLE curation_themes (
    theme_id VARCHAR(50) PRIMARY KEY,               -- 例如 'T001'
    curation_type VARCHAR(50) NOT NULL,             -- 'trend' (時事), 'festival' (節慶), 'custom' (自訂 Prompt)
    title VARCHAR(255) NOT NULL,                    -- 主題名稱
    outline TEXT NOT NULL,                          -- 規劃大綱
    target_audience VARCHAR(255) NOT NULL,          -- 預估受眾
    keywords JSONB NOT NULL,                        -- 當時使用的關鍵字清單 (Array of strings)
    prompt TEXT NULL,                               -- 自訂 Prompt 內容
    year INTEGER NULL,                              -- 節慶指定年份
    created_by UUID REFERENCES users(id),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- 4. 策展企劃書草案表
CREATE TABLE proposals (
    proposal_id VARCHAR(50) PRIMARY KEY,            -- 例如 'P001'
    theme_id VARCHAR(50) REFERENCES curation_themes(theme_id) ON DELETE SET NULL,
    title VARCHAR(255) NOT NULL,
    content TEXT NOT NULL,                          -- 企劃書完整 HTML 格式內容 (所見即所得編輯器編輯用)
    matched_books JSONB NULL,                       -- 匹配成功之圖書清單 (含書名、ISBN、分類號)
    status VARCHAR(50) NOT NULL DEFAULT 'draft',    -- 'draft' (草稿), 'completed' (已確認), 'exported' (已匯出)
    created_by UUID REFERENCES users(id),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- 5. 系統效益日誌表 (計算戰情室數據使用)
CREATE TABLE cost_benefit_logs (
    log_id SERIAL PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    action VARCHAR(100) NOT NULL,                  -- 'theme_generation', 'proposal_export'
    time_saved_hours NUMERIC(5, 2) NOT NULL,       -- 省下工時
    cost_saved_amount NUMERIC(10, 2) NOT NULL,     -- 省下經費
    timestamp TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- 6. 系統參數設定表
CREATE TABLE system_settings (
    setting_key VARCHAR(100) PRIMARY KEY,          -- 'hourly_rate', 'base_theme_hours', 'base_proposal_hours'
    setting_value VARCHAR(255) NOT NULL,
    description TEXT NULL
);

-- 預設參數寫入
INSERT INTO system_settings (setting_key, setting_value, description) VALUES
('hourly_rate', '200', '館員平均時薪 (NTD)'),
('base_theme_hours', '4', '傳統人工主題發想所需工時 (小時)'),
('base_proposal_hours', '16', '傳統人工撰寫企劃書所需工時 (小時)')
ON CONFLICT (setting_key) DO NOTHING;
```

## 4. API 設計規格

所有 API 統一使用 JSON 格式進行資料交換，並基於 `/curation_management/backend` 為前綴。

### API 1: AI 智慧策展發想主題生成
- **Method**: `POST`
- **Path**: `/curation_management/backend/generate_themes`
- **Request Body**:
  ```json
  {
    "curation_type": "trend",
    "keywords": ["AI人工智慧", "ChatGPT", "數位轉型"],
    "prompt": "自訂提示詞內容 (可為空)",
    "year": 2026
  }
  ```
- **Response Body (201 Created)**:
  ```json
  {
    "status": "success",
    "data": [
      {
        "theme_id": "T001",
        "title": "迎接 AI 浪潮：圖書館的數位未來",
        "outline": "展區規劃包括：AI 發展史、AI 在各產業的應用，以及我們與 AI 共生的未來書籍特展。",
        "target_audience": "全體市民與大專院校學生"
      }
    ]
  }
  ```

### API 2: 拋轉至企劃管理中心 (建立企劃草案)
- **Method**: `POST`
- **Path**: `/curation_management/backend/export_to_proposal`
- **Request Body**:
  ```json
  {
    "theme_id": "T001",
    "title": "迎接 AI 浪潮：圖書館的數位未來",
    "outline": "展區規劃包括：AI 發展史、AI 在各產業的應用，以及我們與 AI 共生的未來書籍特展。",
    "target_audience": "全體市民與大專院校學生"
  }
  ```
- **Response Body (201 Created)**:
  ```json
  {
    "status": "success",
    "proposal_id": "P001",
    "message": "已成功建立企劃書草案，請至企劃管理中心編輯。"
  }
  ```

### API 3: 查詢歷史發想紀錄
- **Method**: `GET`
- **Path**: `/curation_management/backend/history`
- **Parameters**: `cur_page=1`, `size=10`, `curation_type=trend` (可選)
- **Response Body (200 OK)**:
  ```json
  {
    "total": 15,
    "data": [
      {
        "theme_id": "T001",
        "curation_type": "trend",
        "title": "迎接 AI 浪潮：圖書館的數位未來",
        "create_time": "2026-06-15 12:00:00"
      }
    ]
  }
  ```

### API 4: 批次導入館藏檔案 (由 DB 解析，Web Service 提供接口)
- **Method**: `POST`
- **Path**: `/curation_management/backend/catalog/import`
- **Content-Type**: `multipart/form-data`
- **Form Data**:
  - `file`: Excel 或 CSV 檔案 (檔案大小上限 10MB)
- **說明**: 系統接收檔案後，由 DB 呼叫 `CatalogService` 進行資料解析。每一筆書目資料將結合書名與簡介，透過 AI (Gemini Embedding API) 生成 768 維向量，並以 Bulk Insert 方式批次存入 PostgreSQL `catalog_books` 資料表的 `embedding` 欄位中。
- **Response Body (200 OK)**:
  ```json
  {
    "status": "success",
    "imported_count": 1250,
    "message": "已成功解析並批次生成向量寫入 1250 筆館藏圖書資料"
  }
  ```

### API 5: 查詢/保存企劃書內文與館藏匹配
- **Method**: `GET` / `PUT`
- **Path**: `/curation_management/backend/proposals/{proposal_id}`
- **PUT Request Body** (線上編輯器儲存):
  ```json
  {
    "title": "迎接 AI 浪潮：圖書館的數位未來",
    "content": "<h1>企劃書大綱...</h1><p>線上編輯器修改後之 HTML 內文...</p>",
    "status": "completed"
  }
  ```
- **Response Body (200 OK)**:
  ```json
  {
    "status": "success",
    "data": {
      "proposal_id": "P001",
      "title": "迎接 AI 浪潮：圖書館的數位未來",
      "content": "<h1>...</h1>",
      "matched_books": [
        {
          "title": "AI 時代生存指南",
          "isbn": "9789861234567",
          "classification_no": "733.4"
        }
      ],
      "status": "completed"
    }
  }
  ```

### API 6: 匯出企劃書 Word / PDF
- **Method**: `GET`
- **Path**: `/curation_management/backend/proposals/{proposal_id}/export`
- **Parameters**: `format=docx` 或 `format=pdf`
- **Response**: 二進位檔案流 (File Stream)，瀏覽器自動觸發下載。

### API 7: 獲取數據戰情室統計數據
- **Method**: `GET`
- **Path**: `/curation_management/backend/dashboard/stats`
- **Response Body (200 OK)**:
  ```json
  {
    "cumulative_hours_saved": 158.0,
    "cumulative_cost_saved": 31600.0,
    "theme_generation_count": 19,
    "proposal_export_count": 10,
    "monthly_stats": [
      { "month": "2026-06", "hours": 42.0, "cost": 8400.0 }
    ]
  }
  ```

## 6. 數據戰情室效益計算邏輯

### 1. 計算公式
每次使用者使用系統，系統會自動在 `cost_benefit_logs` 中記錄該次操作省下的工時。其計算公式基準如下：
- **AI 智慧發想 (Theme Generation)**：
  - 每次點擊生成：**省下 2 小時** (傳統人工需要收集時事與寫下多個草案需 4 小時，系統可快速輔助完成)。
- **拋轉企劃書與匯出 (Proposal Export & Download)**：
  - 每次成功拋轉並確認匯出：**省下 12 小時** (傳統人工排版及從零撰寫企劃書需 16 小時，系統所見即所得編輯省去大部分工時)。
- **累計省下工時** = `(AI智慧發想次數 * 2) + (企劃書匯出次數 * 12)`
- **累計省下經費** = `累計省下工時 * 系統平均時薪參數` (預設時薪為 NT$ 200)

### 2. 參數調整 (後台管理)
- 管理者 (admin) 可於「系統設定 -> 組織架構與館員權限管理」內之系統設定面板更新參數。
- 變更參數會儲存至 `system_settings`。戰情室在呈現折線圖與計算總節省經費時，應即時讀取最新參數進行運算。
