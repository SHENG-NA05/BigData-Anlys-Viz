# DB 虛擬館藏資料生成設計

本文件規劃 DB 測試用的虛擬館藏資料。目標是產生「看起來像真實圖書館館藏」的資料，讓後續館藏匯入、AI 策展、館藏匹配與儀表板測試可以使用。

## 1. 資料用途

虛擬館藏資料主要用於：

- 測試 `catalog_books` 批次匯入。
- 測試書名、ISBN、分類號查詢。
- 測試 AI 生成策展主題後，是否能匹配相關館藏。
- 測試儀表板與企劃書流程。
- 展示系統時避免使用真實個資或受版權限制的完整資料。

## 2. 生成方式

使用腳本：

```bash
python backend/app/db/generate_fake_catalog.py --count 1000 --output data/fake_catalog.csv
```

參數說明：

| 參數 | 預設值 | 說明 |
| --- | --- | --- |
| `--count` | `1000` | 要產生的書籍筆數 |
| `--output` | `data/fake_catalog.csv` | 輸出 CSV 路徑 |
| `--seed` | `20260615` | 隨機種子，用來重複產生相同資料 |

範例：

```bash
python backend/app/db/generate_fake_catalog.py --count 5000 --output data/fake_catalog_5000.csv
```

## 3. CSV 欄位設計

| 欄位 | 對應 DB 欄位 | 說明 |
| --- | --- | --- |
| `title` | `catalog_books.title` | 書名 |
| `isbn` | `catalog_books.isbn` | 模擬 ISBN-13 |
| `author` | `catalog_books.author` | 作者或編輯小組 |
| `publisher` | `catalog_books.publisher` | 出版社 |
| `publication_year` | `catalog_books.publication_year` | 出版年份 |
| `classification_no` | `catalog_books.classification_no` | 圖書分類號 |
| `summary` | `catalog_books.summary` | 書籍簡介 |
| `category` | 暫無 DB 欄位 | 大分類，方便檢查與測試 |
| `subcategory` | 暫無 DB 欄位 | 細分類，方便檢查與測試 |

目前 DB schema 沒有 `category` 與 `subcategory` 欄位，所以匯入時可以先忽略這兩欄，只用 `classification_no` 作為主要分類依據。如果後續需要在系統上直接篩選大分類，可以再把這兩個欄位加入 `catalog_books`。

## 5. 真實感設計

為了讓資料看起來比較像真實館藏，生成規則包含：

- ISBN 使用 13 碼格式，並計算檢查碼。
- 出版年份落在 2005 到 2026 年。
- 出版社使用台灣常見出版社與圖書館單位名稱。
- 作者使用常見中文姓氏與名字。
- 書名使用「入門」、「完全指南」、「圖解」、「案例研究」、「理論與實務」等常見命名方式。
- 摘要會包含分類、主題、閱讀推廣與策展用途。
- 產生結果使用 UTF-8 with BOM，方便 Excel 開啟不亂碼。

## 6. 建議產生數量

| 使用情境 | 建議筆數 |
| --- | --- |
| 快速測試匯入 | 100 到 500 筆 |
| API 與查詢測試 | 1000 到 5000 筆 |
| 效能測試 | 10000 到 50000 筆 |
| 展示系統 | 3000 到 10000 筆 |

## 7. 後續可擴充

- 加入館藏狀態，例如可借閱、外借中、遺失、館內閱覽。
- 加入館藏位置，例如總館、分館、兒童區、參考區。
- 加入借閱次數，讓儀表板可以做熱門館藏分析。
- 加入多語系書籍，例如英文、日文、韓文館藏。
- 加入更精準的分類號規則。
- 加入 `proposal_books` 測試資料，模擬企劃書與館藏匹配結果。
